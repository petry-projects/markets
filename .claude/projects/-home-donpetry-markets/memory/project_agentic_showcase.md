This project is a showcase for agentic development methodology.

**Why:** Part of the project's purpose is demonstrating what agentic development can accomplish, not just building a product.

**How to apply:** Features that might seem over-engineered for an MVP (e.g., comprehensive audit logging) are acceptable because agentic development keeps implementation cost reasonable. Don't push back on scope purely on effort grounds if the feature is architecturally sound.
